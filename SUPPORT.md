# Support

## Use GitHub Issues For

- factual errors in the style guide
- broken links
- workflow failures
- requests for additional terminology or writing-guidance coverage

Use the provided issue templates so reports include the context needed to act on
them.

## Do Not Use Public Issues For

- vulnerability reports
- sensitive incident details
- credentials, tokens, or private infrastructure information

Follow `SECURITY.md` for security reporting instead.

## General Questions

This repository is documentation, not a product support channel for specific
WordPress sites or hosting environments.

For consulting, environment-specific questions, or follow-up that does not
belong in a public issue, use the contact methods listed on
[Dan Knauss's GitHub profile](https://github.com/dknauss) or
[dan.knauss.ca](https://dan.knauss.ca).
